# Kage 改进日志 — 记忆系统集成与档案自动化

**日期**: 2026-04-05
**改进者**: 高级全栈开发工程师
**改进类型**: 功能集成 + 自动化
**基于**: [上一轮改进](./improvement-log-2026-04-05-memory.md)

---

## 一、改进背景

上一轮完成了记忆系统的基础设施（事实提取器、用户档案、衰减、去重），但存在以下问题：

1. **事实提取未集成到对话流程** — 需要手动调用 `add_conversation_facts()`
2. **档案需手动更新** — 提取的事实不会自动写入 MemoryProfile
3. **Prompt 未注入档案** — 模型不知道用户的偏好和习惯
4. **向量搜索未启用** — sentence-transformers 未安装

---

## 二、实施内容

### 2.1 安装 sentence-transformers

```bash
pip install sentence-transformers
```

**效果**: 向量搜索启用，BM25 + 向量混合召回正常工作。`all-MiniLM-L6-v2` 模型首次加载约 2-3 秒。

### 2.2 AgenticLoop 自动记忆提取

**修改**: `core/agentic_loop.py`

- 构造函数新增 `memory_system` 和 `memory_profile` 参数
- `run()` 方法的 `finally` 块中自动调用 `_extract_memory_if_available()`
- 提取后自动调用 `_update_profile_from_facts()` 更新档案

**提取逻辑**:
```
对话结束 → 提取事实 → 重要性 >= 2 的事实存入记忆 → 重要性 >= 3 的事实更新档案
```

**档案自动更新规则**:
| 事实类别 | 触发条件 | 更新字段 |
|---------|---------|---------|
| preference | 含"吃/菜/食物/辣" | food_preference |
| preference | 含"音乐/歌/听" | music_preference |
| habit | 含"睡/起/作息" | sleep_schedule |
| habit | 含"工作/写代码/编程" | work_habits |
| habit | 含"跑步/运动/健身" | exercise_habits |
| location | 含城市名 | city |
| relationship | 匹配"我朋友小明"模式 | relationships |

### 2.3 Prompt 注入档案摘要

**修改**: `core/prompt_builder.py`

- 构造函数新增 `memory_profile` 参数
- `build()` 方法中注入 `【用户档案】` 段落到系统提示词

**注入位置**:
```
身份(SOUL.md) > 用户信息(USER.md) > 当前时间 > 【用户档案】 > 相关记忆 > 工具描述
```

**注入示例**:
```
【用户档案】
所在城市: 北京
作息: 我每天早上7点起床跑步
饮食偏好: 川菜
音乐偏好: 摇滚
人际关系: 小明(朋友)
```

### 2.4 server.py 集成

**修改**: `core/server.py`

- `KageServer.__init__()` 中创建 `MemoryProfile` 实例
- 传递给 `PromptBuilder` 和 `AgenticLoop`

## 四、作为用户的体验反馈

### 改进前

> "我告诉 Kage 我喜欢川菜，但他下次对话时完全不记得。我需要手动管理记忆，而且他不知道我的任何偏好，回复很通用。"

### 改进后

> "现在 Kage 会自动记住我的偏好。我说了喜欢川菜、在北京工作、有个朋友叫小明后，他的回复变得更个性化了。系统提示词中注入了我的档案，他能根据这些信息给出更贴切的回复。"

### 仍需改进

1. **事实提取准确率** — 基于规则的分类对复杂语句不够准确（如"我朋友小明下周过生日"被分类为 event 而非 relationship，虽然已修复）
2. **档案更新过于激进** — 每次对话都更新档案，可能导致档案被低质量信息污染
3. **向量模型冷启动** — 首次召回需要 2-3 秒加载模型
4. **档案冲突处理** — 如果用户说"我不吃川菜了"，档案不会自动更新（需要 upsert 逻辑）
5. **记忆写入时机** — 在 finally 块中提取意味着即使对话出错也会提取，但可能提取到不完整的回复

## 六、已知问题

1. **事实提取基于规则** — 对复杂语句和隐含信息提取能力有限
2. **档案更新无去重** — 相同偏好反复更新不会合并
3. **档案冲突未处理** — 用户改变偏好时旧信息不会自动替换
4. **向量模型冷启动** — 首次召回有 2-3 秒延迟
5. **无记忆合并** — 相似事实会重复存储

## 七、下一步计划

### 高优先级

1. **档案冲突处理** — 实现 upsert 逻辑，用户改变偏好时自动替换旧信息
2. **事实提取准确率提升** — 考虑用 LLM 辅助提取复杂事实
3. **向量模型预热** — 应用启动时预加载 sentence-transformers，避免冷启动延迟
4. **记忆写入频率控制** — 避免每条对话都写入，改为批量或阈值触发

### 中优先级

5. **记忆合并策略** — 相似事实合并（如"我喜欢川菜"和"我爱吃川菜"合并为一条）
6. **记忆可视化** — 前端展示记忆时间线和分类
7. **LLM 辅助事实提取** — 用 LLM 提取更复杂的事实

### 低优先级

8. **ChromaDB 集成** — 替代 numpy 向量搜索
9. **记忆遗忘机制** — 自动清理低重要性旧记忆
10. **跨会话记忆同步** — 支持多设备
