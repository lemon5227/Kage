## 一、改进背景

### 问题诊断

作为用户测试 Kage 时，发现记忆系统存在以下问题：

1. **所有记忆重要性都是 1** — 没有区分重要和无意义的对话
2. **无事实提取** — 存储原始对话而非结构化事实
3. **无去重机制** — 重复内容反复存储
4. **无衰减策略** — 旧记忆和新记忆权重相同
5. **召回排序 Bug** — 先按相关性选出 top_k，最后又按重要性重新排序，导致不相关的高重要性记忆排第一

### 用户场景痛点

**场景**: 用户说"我喜欢吃川菜"
- **改进前**: 存储为 importance=1 的原始对话，召回时被其他高重要性但不相关的记忆覆盖
- **改进后**: 提取为 preference 类别的结构化事实，importance=4，召回时正确排第一

## 三、技术细节

### 3.1 事实提取器 (MemoryExtractor)

**设计思路**: 使用规则引擎 + 正则表达式进行快速分类和重要性评估

**分类体系**:
- `preference` — 用户偏好（喜欢/讨厌/偏好）
- `habit` — 用户习惯（每天/经常/作息）
- `relationship` — 人际关系（朋友/同事/家人）
- `event` — 重要事件（考试/面试/生日）
- `location` — 位置信息（城市/工作地点）
- `other` — 其他

**重要性评估 (1-5)**:
- 基础分: 1
- 类别加成: preference/relationship +2, habit/event +1
- 情感强度: 强烈情感词 +1
- 用户直接陈述: +1
- 长度 > 20 字符: +1

**无意义过滤**:
- 单字/双字回复（嗯嗯、好的）
- 无实质内容的短句

**测试结果**: 10/10 通过

### 3.2 用户档案 (MemoryProfile)

**设计思路**: 结构化 JSON 存储，支持 upsert 操作

**档案字段**:
- 基本信息: name, city, occupation, timezone
- 偏好: food, music, work_habits
- 习惯: sleep_schedule, work_schedule, exercise_habits
- 人际关系: relationships (name, relationship, notes)
- 重要日期: important_dates (date, event, reminder)

**用途**: 生成 prompt 上下文，让 Kage 了解用户

### 3.3 记忆衰减 (recall_with_decay)

**算法**: 指数衰减
```
decayed_importance = importance * 0.5^(age_days / decay_days)
```

**效果**:
- 1 天前的记忆: 衰减系数 ~0.98
- 30 天前的记忆: 衰减系数 0.5
- 60 天前的记忆: 衰减系数 0.25
- 90 天前的记忆: 衰减系数 0.125

**测试结果**: ✅ 新记忆优先级高于旧记忆

### 3.4 记忆去重 (deduplicate_memories)

**策略**:
1. 优先使用向量相似度 (cosine similarity >= 0.85)
2. 回退到 BM25 分数或精确匹配

**测试结果**: ✅ 正确识别重复内容

### 3.5 召回排序 Bug 修复

**Bug**: `recall()` 方法先按 final_scores 选出 top_k，最后又执行 `results.sort(key=lambda x: x["importance"], reverse=True)`，完全覆盖了相关性计算。

**修复**: 移除最后的 importance 排序，保持 final_scores 排序。final_scores 已经是 85% 相关性 + 15% 重要性的加权结果。

**BM25 归一化修复**:
- 问题: BM25 对短文本+单字分词产生负分数
- 解决: 当所有 BM25 分数为负时，回退到关键词重叠评分

## 五、作为用户的体验反馈

### 改进前

> "Kage 记不住我说过什么。我告诉他我喜欢川菜，下次问的时候他好像完全不记得。而且他什么都记，连'嗯嗯'、'好的'这种废话都存起来，检索出来的结果乱七八糟。"

### 改进后

> "现在 Kage 能记住我的偏好了。我告诉他我喜欢川菜、在北京工作、有个朋友叫小明，之后问相关话题时他能正确召回这些信息。而且他不会把无意义的对话存进来，记忆质量明显提高了。"

### 仍需改进

1. **向量模型未加载** — sentence-transformers 未安装，目前只使用 BM25 + 关键词重叠
2. **事实提取准确率** — 基于规则的分类对复杂语句可能不够准确，未来可考虑用 LLM 辅助提取
3. **档案自动更新** — 目前档案需要手动更新，未来应该从提取的事实中自动更新档案
4. **记忆写入时机** — 目前每次对话都提取事实，应该只在对话结束后批量提取

## 六、下一步计划

### 高优先级

1. **安装 sentence-transformers** — 启用向量搜索，提升召回质量
2. **集成到 AgenticLoop** — 在对话结束后自动调用 `add_conversation_facts()`
3. **档案自动更新** — 从提取的事实中自动更新 MemoryProfile
4. **Prompt 注入** — 在构建 prompt 时注入档案摘要

### 中优先级

5. **LLM 辅助事实提取** — 用 LLM 提取更复杂的事实（如"我老板是个傻逼" → relationship: 老板, sentiment: negative）
6. **记忆合并策略** — 相似事实合并（如"我喜欢川菜"和"我爱吃川菜"合并为一条）
7. **记忆可视化** — 前端展示记忆时间线和分类

### 低优先级

8. **ChromaDB 集成** — 替代 numpy 向量搜索，支持更大规模记忆
9. **记忆遗忘机制** — 自动清理低重要性旧记忆
10. **跨会话记忆** — 支持多设备/多会话的记忆同步
