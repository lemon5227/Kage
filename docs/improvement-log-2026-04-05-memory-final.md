## 二、实施内容

### 2.1 档案版本历史

**新增**: `MemoryProfile` 版本历史功能

- `_save_version_history()` — 每次保存前自动备份当前版本
- `get_version_history()` — 获取可用版本列表
- `restore_version(version)` — 恢复指定版本

**存储结构**:
```
~/.kage/memory/
├── profile.json          # 当前档案
└── history/
    ├── v1.json           # 版本 1
    ├── v2.json           # 版本 2
    └── v3.json           # 版本 3
```

### 2.2 记忆遗忘机制

**新增**: `MemorySystem.forget_old_memories()`

智能遗忘策略：
- **永远保留**: 重要性 >= 4 的记忆（极高重要性）
- **近期保留**: 7 天内的所有记忆
- **自动遗忘**: 超过 90 天且重要性 < 2 的记忆
- **条件遗忘**: 超过 7 天且重要性 < 2 的记忆

**效果**: 记忆库保持精简，重要信息永不丢失。

### 2.3 API 端点扩展

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/memory/profile/history` | GET | 获取档案版本历史 |
| `/api/memory/profile/restore/{version}` | POST | 恢复档案版本 |
| `/api/memory/forget` | POST | 执行记忆遗忘 |

## 五、作为用户的最终体验

### 完整用户旅程

1. **首次对话**: Kage 启动快（异步预热），对话流畅
2. **透露偏好**: "我喜欢吃川菜" → 自动提取为 preference 事实
3. **后续对话**: Kage 回复更个性化，因为档案已注入 prompt
4. **改变偏好**: "我不吃川菜了" → 档案自动更新，版本历史自动记录
5. **查看记忆**: 通过前端 API 查看和管理记忆
6. **清理记忆**: 一键去重、合并相似记忆、遗忘旧记忆
7. **回溯档案**: 查看档案变更历史，恢复到任意版本

### 记忆系统架构总览

```
用户对话
  │
  ├─ AgenticLoop.run()
  │     │
  │     ├─ LLM 提取 (优先)
  │     ├─ 规则提取 (回退)
  │     ├─ 档案自动更新 (upsert + 否定检测 + 版本历史)
  │     └─ 批量写入 (每 3 个事实)
  │
  ├─ MemorySystem.recall(query)
  │     ├─ BM25 (关键词)
  │     ├─ 向量 (语义)
  │     ├─ 混合加权 (85% 相关 + 15% 重要)
  │     └─ 时间衰减
  │
  ├─ MemorySystem.merge_similar_facts()
  │     └─ 向量相似度分组 + 整合上下文
  │
  ├─ MemorySystem.forget_old_memories()
  │     ├─ 保留: 重要性 >= 4 (永不遗忘)
  │     ├─ 保留: 7 天内 (近期记忆)
  │     └─ 遗忘: > 90 天且重要性 < 2
  │
  └─ API 端点
        ├─ GET  /api/memory/stats
        ├─ GET  /api/memory/entries
        ├─ POST /api/memory/deduplicate
        ├─ POST /api/memory/merge
        ├─ POST /api/memory/forget
        ├─ GET  /api/memory/profile
        ├─ GET  /api/memory/profile/history
        ├─ POST /api/memory/profile/restore/{version}
        └─ DELETE /api/memory/entries/{id}
```
