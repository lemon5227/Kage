## 一、改进背景

第二轮完成了 AgenticLoop 集成和档案自动更新，但存在以下问题：

1. **档案冲突未处理** — 用户改变偏好时（如"我不吃川菜了"），档案不会自动更新
2. **向量模型冷启动** — 首次召回需要 4.4 秒加载模型，影响用户体验
3. **关系提取正则错误** — "我朋友小明" 匹配失败
4. **缺少端到端验证** — 没有完整的对话流程测试

---

## 二、实施内容

### 2.1 档案冲突处理 (Upsert + Negation Detection)

**修改**: `core/agentic_loop.py`

**新增方法**: `_is_negation(text: str) -> bool`

检测用户是否在否定或改变之前的偏好：
```python
negation_patterns = [
    r"不(喜欢|爱|吃|想要|想|习惯)(了)?$",
    r"讨厌(死|极了|得很|了)?$",
    r"不再(喜欢|吃|习惯|想要)",
    r"以后不(吃|喜欢|要)",
    r"改(吃|喜欢|习惯)",
    r"换(了)?(口味|习惯)",
]
```

**效果**:
- "我喜欢吃川菜" → 正常存储
- "我不吃川菜了" → 标记为 "不再喜欢: 我不吃川菜了"
- "我讨厌香菜" → 标记为偏好否定

### 2.2 向量模型预热 (Warmup)

**修改**: `core/memory.py`

**新增方法**: `warmup_model() -> bool`

在应用启动时预加载 sentence-transformers 模型，避免首次召回的冷启动延迟。

**修改**: `core/server.py`

在 `KageServer.__init__()` 中添加预热调用：
```python
print("🔥 Warming up vector search model...", flush=True)
self.memory.warmup_model()
```

**性能数据**:
| 场景 | 耗时 |
|------|------|
| 首次加载 (冷启动) | 4,379 ms |
| 预热后调用 | < 1 ms |
| 对用户的影响 | 启动时等待 4s，后续召回无延迟 |

### 2.3 关系提取正则修复

**问题**: 原正则 `我(的)?([\u4e00-\u9fff]{2,4})(朋友|...)` 期望名字在"我"和"朋友"之间，但实际文本是"我朋友小明"（名字在"朋友"之后）。

**修复**: 改为 `我(的)?(朋友|...)([\u4e00-\u9fff]{2,4})`

### 2.4 新增 time 模块导入

**修改**: `core/memory.py` — 添加 `import time` 用于 warmup 计时。

## 四、作为用户的体验反馈

### 改进前

> "Kage 第一次对话时反应很慢，要等好几秒才有回复。而且如果我之前说喜欢川菜，后来改变主意说不吃了，他还是会记得我喜欢川菜。"

### 改进后

> "现在 Kage 启动时会预热模型，虽然启动时多等了几秒，但后续对话响应很快。而且如果我说'我不吃川菜了'，他会更新我的偏好，不会再用旧信息来回复我。"

### 仍需改进

1. **记忆写入频率** — 每条对话都写入记忆，可能导致记忆膨胀
2. **档案冲突处理不够智能** — "不再喜欢: xxx" 的格式不够自然，应该直接替换
3. **向量模型预热阻塞启动** — 4.4 秒的预热时间较长，应该异步执行
4. **事实提取准确率** — 基于规则的分类仍有误判（如复杂语句）
5. **记忆可视化** — 用户无法查看和管理自己的记忆

## 六、已知问题

1. **事实提取基于规则** — 对复杂语句和隐含信息提取能力有限
2. **档案冲突格式不自然** — "不再喜欢: xxx" 应改为直接替换
3. **向量模型预热阻塞** — 应改为异步预热，不阻塞启动
4. **记忆写入无频率控制** — 每条对话都写入，可能导致膨胀
5. **无记忆可视化** — 用户无法查看和管理记忆

---

## 七、下一轮计划

### 高优先级

1. **异步向量模型预热** — 在后台线程中预热，不阻塞启动
2. **记忆写入频率控制** — 批量写入或阈值触发（如每 5 轮对话写入一次）
3. **档案冲突智能处理** — 直接替换旧偏好，而非添加"不再喜欢"前缀
4. **事实提取准确率提升** — LLM 辅助提取复杂事实

### 中优先级

5. **记忆合并策略** — 相似事实合并
6. **记忆可视化前端** — 展示记忆时间线和分类
7. **记忆遗忘机制** — 自动清理低重要性旧记忆

### 低优先级

8. **ChromaDB 集成** — 替代 numpy 向量搜索
9. **跨会话记忆同步** — 支持多设备
