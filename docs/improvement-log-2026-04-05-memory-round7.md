## 一、改进背景

第六轮完成了 LLM 辅助提取，但记忆系统仍存在以下问题：

1. **相似事实重复存储** — "我喜欢川菜" 和 "我爱吃川菜" 作为两条独立记忆
2. **无可视化接口** — 前端无法查看和管理记忆
3. **无管理端点** — 无法删除、合并、去重记忆
4. **无档案查询 API** — 无法获取用户档案摘要

## 二、实施内容

### 2.1 记忆合并策略

**新增**: `MemorySystem.merge_similar_facts(similarity_threshold=0.75) -> int`

与去重不同，合并不只是删除重复项，而是：
- 使用向量相似度分组相似事实
- 保留最高重要性的条目作为基础
- 合并上下文信息
- 更新重要性为最大值
- 标记 `merged_from` 字段记录合并来源数

**与去重的区别**:
| 操作 | 行为 | 适用场景 |
|------|------|---------|
| 去重 | 删除完全相同或极相似的条目 | 完全重复 |
| 合并 | 保留一条，整合上下文 | 相似但不同 |

### 2.2 记忆可视化 API

**新增端点**:

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/memory/stats` | GET | 记忆系统统计 |
| `/api/memory/entries` | GET | 列出记忆条目（分页/过滤） |
| `/api/memory/entries/{id}` | DELETE | 删除记忆条目 |
| `/api/memory/deduplicate` | POST | 去重记忆 |
| `/api/memory/merge` | POST | 合并相似记忆 |
| `/api/memory/profile` | GET | 获取用户档案 |

### 2.3 API 响应示例

**GET /api/memory/stats**:
```json
{
    "total_entries": 42,
    "vector_ready": true,
    "bm25_ready": true,
    "embeddings_count": 42
}
```

**GET /api/memory/entries?limit=10&offset=0&category=preference**:
```json
{
    "total": 42,
    "offset": 0,
    "limit": 10,
    "entries": [
        {
            "id": "uuid",
            "timestamp": "2026-04-05T...",
            "content": "我喜欢吃川菜",
            "type": "fact:preference",
            "importance": 4
        }
    ]
}
```

## 五、作为用户的最终体验

### 完整用户旅程

1. **首次对话**: Kage 启动快（异步预热），对话流畅
2. **透露偏好**: "我喜欢吃川菜" → 自动提取为 preference 事实
3. **后续对话**: Kage 回复更个性化，因为档案已注入 prompt
4. **改变偏好**: "我不吃川菜了" → 档案自动更新
5. **查看记忆**: 通过前端 API 查看和管理记忆
6. **清理记忆**: 一键去重和合并相似记忆

### 记忆系统架构总览

```
用户对话
  │
  ├─ AgenticLoop.run()
  │     │
  │     ├─ LLM 提取 (优先)
  │     ├─ 规则提取 (回退)
  │     ├─ 档案自动更新 (upsert + 否定检测)
  │     └─ 批量写入 (每 3 个事实)
  │
  ├─ MemorySystem.recall(query)
  │     ├─ BM25 (关键词)
  │     ├─ 向量 (语义)
  │     ├─ 混合加权 (85% 相关 + 15% 重要)
  │     └─ 时间衰减
  │
  ├─ MemorySystem.merge_similar_facts()
  │     └─ 向量相似度分组 + 整合上下文
  │
  └─ API 端点
        ├─ GET /api/memory/stats
        ├─ GET /api/memory/entries
        ├─ POST /api/memory/deduplicate
        ├─ POST /api/memory/merge
        ├─ GET /api/memory/profile
        └─ DELETE /api/memory/entries/{id}
```
