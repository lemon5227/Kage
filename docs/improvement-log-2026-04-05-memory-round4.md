## 一、改进背景

第三轮完成了冲突检测和向量模型预热，但存在以下问题：

1. **向量模型预热阻塞启动** — 4.4 秒的同步预热导致启动变慢
2. **记忆写入过于频繁** — 每条对话都写入，I/O 开销大
3. **档案冲突格式不自然** — "不再喜欢: xxx" 格式生硬
4. **缺少关闭时的数据刷新** —  pending facts 可能在进程退出时丢失

---

## 二、实施内容

### 2.1 异步向量模型预热

**修改**: `core/server.py`

将同步预热改为后台线程异步执行：
```python
import threading
def _warmup():
    self.memory.warmup_model()
threading.Thread(target=_warmup, daemon=True, name="memory-warmup").start()
```

**效果**: 启动不再阻塞，向量模型在后台加载。首次召回可能稍慢，但不影响启动体验。

### 2.2 记忆写入频率控制 (Batch Writing)

**修改**: `core/agentic_loop.py`

新增批量写入机制：
- `_pending_facts`: 累积待写入的事实
- `_fact_batch_size = 3`: 每 3 个事实触发一次持久化
- `flush_pending_facts()`: 强制刷新（用于关闭时调用）

**效果**: 减少 I/O 频率，从每条对话写入改为每 3 条事实写入一次。

### 2.3 档案冲突智能处理

**修改**: `core/agentic_loop.py`

改进否定检测后的档案更新逻辑：
- 提取用户不再喜欢的具体内容
- 使用 "不喜欢: xxx" 格式替代 "不再喜欢: 完整句子"
- 更自然、更易读的档案格式

**示例**:
- 输入: "我不吃川菜了"
- 旧格式: "不再喜欢: 我不吃川菜了"
- 新格式: "不喜欢: 川菜"

## 四、作为用户的体验反馈

### 改进前

> "Kage 启动时要等好几秒才能加载完模型。而且每次说话都会写入记忆，感觉有点浪费资源。如果我说'我不吃川菜了'，档案里会显示'不再喜欢: 我不吃川菜了'，看起来很奇怪。"

### 改进后

> "现在 Kage 启动很快，向量模型在后台加载。记忆写入更智能了，不会每条对话都写。档案格式也更自然了，'不喜欢: 川菜' 比之前的格式好多了。"

### 仍需改进

1. **首次召回延迟** — 异步预热意味着第一次召回可能仍然较慢（如果预热未完成）
2. **pending facts 丢失风险** — 如果进程异常退出，pending facts 可能丢失
3. **事实提取准确率** — 基于规则的分类仍有误判
4. **记忆可视化** — 用户无法查看和管理自己的记忆
5. **档案版本历史** — 没有记录档案变更历史，无法回溯
